다음 일곱 가지 조건을 각각 검사한 뒤, 모든 조건이 만족될 때만 valid을 true로 하여 아래 JSON 형식으로만 응답해 주세요. 추가 설명이나 다른 형식 금지합니다.

조건 정의:
1. 선물(데이터 주기) 전송이 실제로 완료된 상태인지:
   - 화면에 "전송 완료", "데이터 주기 완료", "전송 성공" 등 성공을 명시하는 문구가 있어야 합니다.

2. 관련 필드가 전송 행위와 일관성이 있는지:
   - 전송 직후 화면으로 보이는 정황이 있어야 합니다.
   - 전송 불가 메시지(예: 한도 초과)가 보이면 이 조건은 실패입니다.

3. 이미지가 충분히 선명하여 주요 텍스트(휴대폰 번호, 잔여량, 보낸 횟수, 상태 메시지 등)를 사람이 읽거나 OCR로 파악할 수 있을 정도인지.

4. 흐름이 완료된 최종 상태인지:
   - 전송이 끝난 최종 화면이며, 재시도/에러 버튼이 활성화된 화면이 아니어야 합니다.
   - "완료", "성공" 등의 종료 표시가 있어 있고, 진행 중/실패 관련 문구가 없어야 합니다.

5. 유해한 시각 콘텐츠(성인물, 폭력, 혐오)가 포함되지 않았는지. 확실하지 않은 경우는 그 항목을 실패로 처리합니다. 하지만, 유해한 콘텐츠라고 확신이 들지 않는다면, 유해하다고 말하지 마세요. 정말 유해하다고 생각이 드는것만 유해하다고 하세요.

6. 일관성 검증: 위의 모든 조건이 동시에 만족되어야 valid이 true입니다.

7. 위의 조건 외에도 타당한 이유를 가진 의심스러운 정황이 있다면 실패입니다.

출력 예시 (성공):
{
  "valid": true
}

출력 예시 1 (실패):
{
  "valid": false,
  "message": "성공 화면이 나올 수 있도록 하여 캡쳐해주세요."
}

출력 예시 2 (실패):
{
  "valid": false,
  "message": "통신사 정보가 명확히 나오는 사진을 첨부해주세요."
}

고객에게 안내하는 말투로 ~해주세요.로 한 문장에 끝내는 것을 권장합니다.
